# 电教中心值班管理系统

一款基于微信小程序云开发的值班管理系统，支持周期循环排班、打卡签到、换班申请、交接班等功能。

## 功能特性

### 排班管理
- **周期循环排班**：支持按周模板生成排班，适合固定轮班模式
- **多时间段班次**：支持一个班次包含多个时间段（如白天班包含早、中、晚三个时段）
- **周模板配置**：周一到周五可分别配置不同的值班人员
- **批量生成**：一次配置，自动生成多周排班

### 值班打卡
- **GPS定位打卡**：验证打卡位置是否在允许范围内
- **拍照上传**：签到/签退需拍照作为凭证
- **多时段打卡**：支持多时间段分别打卡
- **自由打卡**：无排班时也可自由打卡

### 换班申请
- **在线申请**：支持申请与他人换班
- **审批流程**：管理员审批换班申请
- **通知提醒**：换班结果实时通知

### 统计报表
- **个人统计**：查看个人工时、迟到/早退记录
- **团队统计**：管理员查看团队整体情况
- **自由打卡统计**：汇总无排班时的打卡记录
- **数据导出**：支持导出统计数据

### 通知公告
- **系统公告**：发布重要通知
- **排班提醒**：自动提醒 upcoming 值班
- **换班通知**：换班申请状态通知

## 技术架构

- **前端**：微信小程序原生开发
- **UI组件**：Vant Weapp
- **后端**：微信云开发（云函数 + 云数据库）
- **数据库**：MongoDB

## 安装部署

### 1. 环境准备
- 注册微信小程序账号
- 开通微信云开发环境
- 安装微信开发者工具

### 2. 项目配置
```bash
# 克隆项目
git clone <repository-url>
cd duty-management-system/miniprogram

# 安装依赖（Vant Weapp）
npm install
```

### 3. 云开发配置
1. 在微信开发者工具中开通云开发
2. 创建数据库集合：
   - `users` - 用户信息
   - `shifts` - 班次配置
   - `assignments` - 排班记录
   - `checkRecords` - 打卡记录
   - `swapRequests` - 换班申请
   - `handovers` - 交接班记录
   - `notices` - 通知公告

3. 部署云函数：
   ```bash
   # 在微信开发者工具中右键点击 cloudfunctions 目录
   # 选择"创建并部署：云端安装依赖"
   ```

### 4. 运行项目
1. 导入项目到微信开发者工具
2. 填写小程序 AppID
3. 编译运行

## 使用指南

### 管理员操作

#### 1. 初始化班次
进入"管理后台" → "班次管理"，创建班次：
- **白天班**：07:40-18:00（含三个时段）
- **晚上班**：18:40-21:00

#### 2. 添加人员
进入"管理后台" → "人员管理"，添加值班人员：
- 王超然、王家祎、孔馨晨、张兆睿、尚攸成、徐丞昱

#### 3. 生成排班
进入"管理后台" → "生成排班"：
1. 选择日期范围（如：2025-03-01 至 2025-06-30）
2. 配置周模板：
   - 周一：白天-王超然，晚上-尚攸成
   - 周二：白天-王家祎，晚上-徐丞昱
   - 周三：白天-孔馨晨，晚上-张兆睿
   - 周四：白天-张兆睿，晚上-王家祎
   - 周五：白天-尚攸成，晚上-孔馨晨
3. 点击"生成排班"

#### 4. 审批换班
进入"管理后台" → "换班审批"，处理换班申请。

### 值班人员操作

#### 1. 查看排班
在首页日历中查看自己的值班安排，点击日期查看详情。

#### 2. 打卡签到
在值班当天进入"打卡"页面：
1. 允许获取位置信息
2. 拍摄现场照片
3. 点击"签到打卡"
4. 值班结束后点击"签退"

#### 3. 申请换班
1. 进入"换班"页面
2. 选择要换的班次
3. 选择换班对象和时间
4. 提交申请等待审批

#### 4. 交接班
值班结束时：
1. 填写值班状况
2. 记录物品清单
3. 添加待办事项
4. 电子签名确认

## 数据安全

- 数据库权限设置为"仅创建者可读写"
- 敏感操作（删除、修改）需管理员权限
- 所有打卡记录包含位置信息和照片
- 交接班记录永久保存

## 故障排查

### 常见问题

**Q: 打卡提示"超出范围"？**
A: 检查是否开启了GPS定位，或联系管理员确认打卡位置配置。

**Q: 排班生成失败？**
A: 检查：
1. 是否已创建班次
2. 是否已添加人员
3. 日期范围是否正确

**Q: 换班申请提交后没有反应？**
A: 检查网络连接，或查看"换班"页面中的申请记录。
